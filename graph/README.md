# Research Knowledge Graph

A living semantic graph of research concepts extracted from publications,
visualised as the background topology of the personal website.

## Structure

```
graph/
├── graph.json        ← canonical data (commit this to git)
├── editor.html       ← visual editor (open in browser)
├── README.md
└── tool/
    ├── index.js      ← CLI
    ├── package.json
    ├── .env.example
    └── sources/      ← drop PDFs here
```

## Setup

```bash
cd graph/tool
cp .env.example .env
# Edit .env — add your ANTHROPIC_API_KEY
# ORCID_ID is pre-filled as 0000-0003-4122-6041
npm install
```

## Commands

### Pull your publication list from ORCID
```bash
node index.js orcid
# Shows all ORCID works, marks which are in graph, prints add commands for missing ones
```

### Search for papers
Searches PubMed, arXiv, and Semantic Scholar in parallel.
```bash
node index.js search "bayesian surprise creativity"
node index.js search "temporal discounting reward" --author "Hakimi"
```

### Add a paper
```bash
node index.js add --pdf sources/hakimi2025.pdf       # local PDF (best quality)
node index.js add --doi 10.1145/3411764.3445593       # by DOI
node index.js add --url https://arxiv.org/abs/2310.x  # arXiv URL
node index.js add --pmid 38041234                     # PubMed ID
node index.js add --text                              # paste via stdin
node index.js add --pdf sources/paper.pdf --dry-run  # preview only
```

### Other
```bash
node index.js list      # all papers with nodes contributed
node index.js stats     # counts + top nodes by degree
node index.js rebuild   # re-run force layout, save graph.json
```

## Typical workflow
```bash
node index.js orcid                    # see what's missing
node index.js add --doi 10.xxxx/xxx   # add missing papers
node index.js rebuild                  # tidy layout
git add graph/graph.json && git commit -m "add N papers"
```

## Sources used

| Source | Purpose |
|--------|---------|
| ORCID (0000-0003-4122-6041) | Canonical publication list |
| Unpaywall | Open-access PDF by DOI |
| Semantic Scholar | Search + abstract fallback |
| PubMed E-utilities | Abstract by PMID |
| arXiv | Full-text PDF + search |

## graph.json schema

- **weight** — conceptual centrality (0.5 minor → 1.0 core)
- **strength** — theoretical coupling between concepts
- **cluster** — research area (0=Psychology, 1=Bayesian, 2=AI, 3=Application, 4=Neuroeconomics, 5=Social Neuro, 6=Consumer)
- **layout** — pre-computed XY positions; regenerated by `rebuild`
